## Présentation

>Cette API REST permet de gérer les stages de l'application interne du lycée VHB : stages, organisations, contacts.
>
>Les principaux besoins sont :
>- Faire évoluer cette base de données pour que les étudiants puissent eux-mêmes saisir les informations nécessaires à l'élaboration de leur fiche de validation de stage.
>- Proposer une API-Rest concernant les ressources liées aux stages afin d'élargir les applications clientes potentielles (web, mobile).
>
>J'ai été assigné à la partie `stages`, et mes 2 autres collègues de travail se sont occupés de la partie `organisation`, et `contacts`.


### Outils

> - PHP Symfony 7 (sous VSCode)
> - Podman
> - Talend API Tester / Insomnia

### Architecture rapide

> - Entités : représentent les tables (Etudiant, Stage, Organisation, etc.).
> - Controllers : contrôlent avec les routes (GET, POST, PUT).
> - Serializer : conversion Entity <-> JSON.
> - Security : accès par JWT (Json Web Token)

### Ce que j'ai effectué 

> - GET /api/stages — lister tout les stages
> - GET /api/stages/{id} — récupérer un stage
> - POST /api/stages — créer un stage

### Installation et exécution

> - Environnement Local et développement : podman
> - Environnement de production : VM Ubuntu Server 24

### Contrôle des données POST

>Le contrôle des données envoyées dans le payload de la requête POST est également vérifié à l'aide du composant Validator de symfony, avant d'être envoyé sur le serveur pour éviter des problèmes comme des erreurs 500, ou renvoyer une bad request (erreur 400) s'il manque des choses ou des données erronées :
```php
        $constraints = new Assert\Collection(
            [    
                'descriptifMission' => [new Assert\NotBlank(), new Assert\Type(
                    type:'string',
                    message:'descriptifMission n\'est pas de type string',
                )],
                'moyens' => [new Assert\NotBlank(), new Assert\Type(
                    type:'string',
                    message:'moyens n\'est pas de type string',
                )],
                'idOrganisation' => [new Assert\NotBlank(), new Assert\Type(
                    type:'integer',
                    message:'idOrganisation n\'est pas de type integer',
                )],
                'idPeriodeStage' => [new Assert\NotBlank(), new Assert\Type(
                    type:'integer',
                    message:'idPeriodeStage n\'est pas de type integer',
                )],
                'idEtudiant' => [new Assert\NotBlank(), new Assert\Type(
                    type:'integer',
                    message:'idEtudiant n\'est pas de type integer',
                )]
            ]
        );
```

Pour ensuite vérifier les erreurs ett envoyer une JsonReponse Badrequest (400) au client web s'il y a des erreurs :
```php
        $errors = $unValidateur->validate($contenuJson, $constraints);
        if (count($errors) > 0) {
            foreach($errors as $error){
                $msgError[]= $error->getMessage();
            }
            return new JsonResponse(['message' => $msgError], JsonResponse::HTTP_BAD_REQUEST);
        }
```
