# Introduction

### Dans cet atelier de professionnalisation, on nous à confié la tâche de répondre à un besoin pour la platefore gitlab.

>Ce besoin était de pouvoir altérer certaines données de GitLab en masse via un site web.
>
>La solution est de créer cette dite page web permettant de récupérer et d'afficher les données de GitLab par son API, et de pouvoir en altérer les données en fonction des besoins.

>Les fonctions permettant ces appels sur l'API de GitLab sont faites avec le langage de programmation [JavaScript](https://developer.mozilla.org/fr/docs/Web/JavaScript).
>
>Les tests sur cette API ont été faits à l'aide de l'extension Chrome [Talend API Tester](https://chromewebstore.google.com/detail/talend-api-tester-free-ed/aejoelaoggembcahagimdiliamlcdmfm?hl=fr)

Ce besoin est décrit dans le [Cahier Des Charges](https://gitlab.siovhb.lycee-basch.fr/matheo-travers/ap31-gitlab-admin/-/blob/8bf87c5b6607f11bcd9469c8b05e5b1b834e20c2/docs/cdc.md) et peut être résumé par les exigences suivantes :

>#### Projets :
>(Voir si possible [l'issue correspondante](https://gitlab.siovhb.lycee-basch.fr/matheo-travers/ap31-gitlab-admin/-/issues/17))
>- Récupérer les données et les faire apparaître sur le site web solution.
>- Créer une checkbox pour chaque projet permettant de séléctionner X nombre de ces derniers en même temps.
>- Supprimer X nombre projets séléctionnés.
>- Changer la visibilité de X nombre projets séléctionnés.

>#### Utilisateurs :
>(Voir si possible [l'issue correspondante](https://gitlab.siovhb.lycee-basch.fr/matheo-travers/ap31-gitlab-admin/-/issues/18))
>- Récupérer les données et les faire apparaître sur le site web solution.
>- Créer une checkbox pour chaque projet permettant de séléctionner X nombre utilisateurs en même temps.
>- Activer/Désactiver X nombre utilisateurs séléctionnés.
>- Changer la limite de nombre de projets maximum pouvant être créer par X nombre utilisateurs.

>#### Groupes :
>(Voir si possible [l'issue correspondante](https://gitlab.siovhb.lycee-basch.fr/matheo-travers/ap31-gitlab-admin/-/issues/16))
>- Récupérer les données et les faire apparaître sur le site web solution.
>- Créer une checkbox pour chaque projet permettant de séléctionner X nombre groupes en même temps.
>- Supprimer X nombre utilisateurs séléctionnés.
>- Changer le rôle nécessaire pour créer un projet dans le groupe.

Ces différents besoins, en fonction de leur catégorie, sera disposé dans le site final, séparé par catégorie (Projets, Utilisateurs, Groupes).
